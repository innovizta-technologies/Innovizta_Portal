# Business Logic Documentation

This folder contains all business requirements, database schemas, workflows, and relations for the Innovizta ERP system.

## How to Use

1. **Copy** `_template.md` to a new file (e.g., `user-roles.md`, `resource-requests.md`)
2. **Fill in** the sections with your business requirements
3. **Save** and notify the development team
4. Development will implement based on these specifications

## Files Expected from Team

| File | Purpose | Status |
|------|---------|--------|
| `user-roles.md` | User types, permissions, role hierarchy | ⏳ Pending |
| `database-schema.md` | Tables, columns, relationships | ⏳ Pending |
| `workflows.md` | Business processes and workflows | ⏳ Pending |
| `resource-requests.md` | Client request → Worker fulfillment flow | ⏳ Pending |
| `projects.md` | Project structure, milestones, tracking | ⏳ Pending |
| `notifications.md` | Email, in-app notification rules | ⏳ Pending |
| `reports.md` | Reports, dashboards, analytics needed | ⏳ Pending |

## Template Guide

Each MD file should follow this structure:

```markdown
# [Feature Name]

## Overview
What this feature does and why it exists

## Actors
Who uses this feature (roles)

## Rules
Business rules and constraints

## Data
Tables, fields, and relationships involved

## Workflow
Step-by-step process flow

## Edge Cases
Known edge cases or special conditions

## Acceptance Criteria
How we know this feature is complete
```
